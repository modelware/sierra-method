# Sierra Method

![Release](https://img.shields.io/badge/Release-v0.3.0-blue)

This repository provides an [Obsidian](https://obsidian.md) vault (folder) template for modeling and analyzing systems using the **Sierra Method**.

The template is organized based on an [OML](https://github.com/opencaesar/oml) vocabulary for the method. 

## 🚀 Already in Obsidian?

Click on <a href="Sierra Method.canvas" class="internal-link"><button style="color: white; border: 2px solid green; background-color: green; padding: 10px 20px; font-size: 16px; cursor: pointer; border-radius: 5px;">START</button></a>

## 🛠️ Install Obsidian

If you don’t have Obsidian installed, download it from [obsidian.md/download](https://obsidian.md/download).

Install the version for your OS (MacOS, Windows, or Linux)

## 🏁 Clone Repository

Follow the steps below to get up and running with this Obsidian vault.

### 1. Fork the Repository

The preferred approach is for you to fork this repo (and pull it later to get the latest):

1. Click the **Fork** button in the top right corner of this github repo page
2. Choose a GitHub organization or personal account as the destination

This will create your own copy of this repository under your chosen account.

### 2. Clone Your Forked Repository

After forking, you can clone the repository to your local machine:

```bash
git clone https://GITHUB/YOUR_ORGANIZATION/YOUR_FORKED_REPOSITORY.git
```
Replace GITHUB, YOUR_ORGANIZATION, and YOUR_FORKED_REPOSITORY with the actual values from your GitHub account.
## 🧩 **Start Using with Sierra Model**

1. Run [Obsidian](https://obsidian.md) and open the vault at the root of your cloned repository.
	1. Obsidian will remember the vault in the future
 2. In the root, you will see a README file and some other top level folders
	1. Click on the README (this document) then on the green **Start** button
